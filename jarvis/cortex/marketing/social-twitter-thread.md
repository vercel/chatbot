**10 tweets. Publish date: June 17, 2026**

---

**Tweet 1/10:**
Google just published OKF v0.1 — a standard for AI knowledge files.

We read the spec and realized we'd already built it. 6 months ago.

Today we're publishing the superset: Neptune-Knowledge-Spec v1.0.

🧵

**Tweet 2/10:**
OKF defines index.md, log.md, type fields, and cross-links.

Neptune Chat has 500+ knowledge files with all of that — plus 10 innovations Google's spec doesn't cover.

Here's what we built:

**Tweet 3/10:**
1. Playbook Routing — Domain playbooks that route agent intents to skills
2. Agent Skill Definitions — Executable skills with tool manifests + code
3. Mission State Machines — Long-running AI tasks with FSM tracking
4. Cross-Session Memory — Persistent references across conversations

**Tweet 4/10:**
5. Connector Specs — Auto-generate API clients from knowledge files
6. Generative UI — React components bound to connectors
7. Workflow Orchestration — Cron-based automation pipelines
8. Self-Coding — AI modifies its own codebase with guardrails

**Tweet 5/10:**
9. Audit Trails — Structured compliance records
10. Knowledge Graph — D3 force-directed visualization of all files

All of this is additive to OKF. We add fields, never remove. Every NKS file exports as valid OKF.

**Tweet 6/10:**
The spec is live: github.com/abhiswami2121/neptune-chat/docs/NEPTUNE-KNOWLEDGE-SPEC-v1.0.md

Reference implementation running at neptune-chat-ashy.vercel.app/knowledge

**Tweet 7/10:**
We're not competing with OKF. We're augmenting it.

The strategy: OKF is the foundation layer. NKS is the production layer.

Think of it like SQL (standard) vs PostgreSQL (production-grade).

**Tweet 8/10:**
What this means:
- AI agents that execute skills, not just read docs
- Playbooks that route to the right tool for any intent
- Missions tracked through state machines
- Memory that persists across sessions
